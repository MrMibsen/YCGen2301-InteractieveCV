Project: Hotel-reserveringssysteem

Context
Begin 2023 hebben we als traineegroep gewerkt aan een project rondom de ontwikkeling van een hotelreserveringssysteem.

Het project is uitgevoerd via de SCRUM-werkwijze, waarbij we wekelijkse sprints hebben gehanteerd, om zo flexibel om te gaan met de veranderende klantwens.
In dit project bestond de klant uit enkele docenten van de opleider (@TheAcademy).

In vijf weken hebben we een een site met een werkend reserveringssysteem neergezet waar we je graag meer over vertellen!
Kamer boeken bij ons mooie reserveringssysteem

Het proces: van scratch naar product
Vanuit de verschillende projectrollen zijn we gestart met het project. Globaal gezien hebben we de volgende stappen doorlopen:

  • Klantgesprekken, requirements in kaart brengen, wireframes, UML-klassediagram
  • Samenwerking inrichten, o.a. via GitHub en Trello

  • Database: van lokaal opgezet (SQL) naar online deployment
  • Azure Web App (PaaS) - Linux
  • Programmeren via Python en Flask

  • Deployment via Static Web App
  • HTML (inhoud) en CSS (styling) van de site
  • Interactiviteit van de site ingebouwd via JavaScript

  • Mendix, inrichten domain models en microflows
  • Eigen database voor LowCode opgesteld voor hotelreserveringssysteem
  • Via bouwblokken stap voor stap het hotelreserveringssysteem gemaakt

  • Onderzoeksvraag: welke factoren zijn aan te wijzen als annuleringsindicatoren? Hoe kan dit inzichtelijk worden gemaakt?
  • Aanpak: met behulp van algoritmen de factoren achterhalen. Om deze vervolgens verder te onderzoeken.
  • 9 fases doorlopen van data-analyse
  • Aanbevelingen en conclusies
