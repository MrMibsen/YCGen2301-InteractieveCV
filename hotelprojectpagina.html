Manier van (samen)werken
Als projectgroep hebben we gewerkt via de SCRUM-werkwijze met wekelijkse sprints.

Vóór elke sprint werd een planning gemaakt waarin werd bepaald wat er die week gemaakt werd. Na elke sprint was er een sprint-review waarbij de klant zijn mening kon geven over de deelproducten (increments). Op basis hiervan ontstaan nieuwe of veranderende wensen.

De informatie-analisten hadden een coördinerende rol in SCRUM. Denk aan het beheer van het Trello bord, een opzet maken voor de sprint-reviews en het prioriteren van de sprint-backlog.

Samenwerken via Git & GitHub
In het project is samengewerkt via Github en Git. Door op deze wijze samen te werken, werkt elk projectlid aan de meest recente versie.

Hierbij was de coördinatie belangrijk, zodat er geen merge-conflicten ontstonden.
1. Database
De database is lokaal opgezet via MySQL door het creëren van een database via PHP in XAMPP. De database is daarna gekoppeld naar het hotelreserveringssysteem via de software Flask. Het deployen is gebeurd via Microsoft Azure. In de database worden onder andere de boekingen opgeslagen en gekoppeld aan memberaccounts.

2. Azure Web App (PaaS) - Linux
De backend, frontend en database zijn gedeployed via Microsoft Azure. Er is gebruik gemaakt van de Azure Web App, wat zich kenmerkt als een Platform as a Service (PaaS). De deployment is gedaan via een Linux besturingssysteem.

3. Programmeren via Python en Flask
Voor het programmeren van de back-end is gebruik gemaakt van de programmeertaal Python. Denk hierbij aan de de verschillende functionele onderdelen van het hotelreserveringssysteem. Flask is een Python module dat bijdraagt aan het developen van web-applicaties....
