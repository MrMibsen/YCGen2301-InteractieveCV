Meer informatie over het project
Zoals benoemd zijn er meerdere stappen doorlopen tijdens dit project. Op deze pagina wordt een kijkje achter de schermen gegeven door in te zoomen op de back-end en de koppeling van de back-end met de front-end.
We zoomen daarbij in de volgende onderwerpen:

1. Database: van lokaal opgezet (SQL) naar online deployment

2. Frond-end deployement

3. Back-end: programmeren via Python en Flask





Manier van (samen)werken
Als projectgroep hebben we gewerkt via de SCRUM-werkwijze met wekelijkse sprints.

Vóór elke sprint werd een planning gemaakt waarin werd bepaald wat er die week gemaakt ging worden. Na elke sprint was er een sprint-review waarbij de klant zijn mening kon geven over de deelproducten (increments). Op basis hiervan ontstaan nieuwe of veranderende wensen.

De informatie-analisten hadden een coördinerende rol in SCRUM. Denk aan het beheer van het Trello bord, een opzet maken voor de sprint-reviews en het prioriteren van de sprint-backlog.

Samenwerken via Git & Github
In het project is samengewerkt via Github en Git. Door op deze wijze samen te werken, werkt elk projectlid aan de meest recente versie.

Hierbij was de coördinatie belangrijk, zodat er geen merge-conflicten ontstonden.

CI/CD pipeline
Er is een CI/CD pipeline ingesteld. Als er werk via Github gepusht werd naar (en gemerged met) de main branche, werd dit direct gedeployed via Azure. Dit zorgt voor automatisering én zowel efficiënt werken als efficiënt testen tijdens het project.
1. Database
De database was een Azure SQL database. Het beheer van de database ging via MySQL Workbench. De database is daarna gekoppeld naar het hotelreserveringssysteem via communicatie met de backend (via Python en Flask). In de database worden onder andere tabellen met alle boekingen, informatie over de gekozen kamer etc.

2. Front-end - HTML/CSS/Javascript
De front-end is gedeployed door een Static Web App (via HTML/CSS/Javascript). Door de Javascript functies is er data naar de back-end gestuurd, in de back-end is dit aan elkaar gekoppeld in de database. Hierna is er data uit de database teruggestuurd naar de front-end. Denk aan het overzicht met alle boekingen, informatie over de gekozen kamer etc.

3. Back-end - Programmeren via Python en Flask
De backend is gedeployed door de Microsoft Azure Web App wat zich kenmerkt als een Platform as a Service (PaaS). De deployment is gedaan via een Linux besturingssysteem. Voor het programmeren van de back-end is gebruik gemaakt van de programmeertaal Python. Denk hierbij aan de de verschillende functionele onderdelen van het hotelreserverings-
systeem. Flask is een Python module dat bijdraagt aan het developen van web-applicaties.
